Every MP3 file carries hidden metadata — title, artist, album, year, genre. It lives in the first few bytes of the file as structured ID3 frames. This project reads, parses and edits those frames directly using binary file handling in C.

The ID3 Tag Structure

Every MP3 file starts with an ID3v2 header — 10 bytes that describe the tag version, flags and total size. After the header come individual frames, each with a 4-byte ID, 4-byte size and actual data. This project reads and writes those frames directly.

01
Read the ID3 Header
The first 10 bytes of every MP3 file contain the ID3v2 header — "ID3" identifier, version, flags and the total tag size encoded as a syncsafe integer.
fread(&header, 10, 1, fp)
02
Parse Frame by Frame
After the header, each frame has a 4-byte frame ID (like TIT2 for title, TPE1 for artist), a 4-byte size and the actual string data. Frames are read one by one until the tag ends.
fread(&frame, sizeof(FrameHeader), 1, fp)
03
Edit & Write Back
To edit a tag, the new value is written into the correct frame position using fseek() to jump to the exact byte offset. Audio data after the ID3 block is never touched.
fseek(fp, offset, SEEK_SET)

Read & Edit Operations

Two core operations — reading displays all metadata fields from the MP3 file. Editing writes a new value to a specific frame without touching the audio.

READ — DISPLAY METADATA
1
Open MP3 file in binary read mode
fopen(file, "rb")
2
Read and validate 10-byte ID3v2 header
check "ID3" identifier + version
3
Decode syncsafe integer for total tag size
size = (b0<<21)|(b1<<14)|(b2<<7)|b3
4
Loop through frames — read 4-byte ID + size
fread(&frame_id, 4, 1, fp)
5
Read frame data and display field value
fread(buffer, frame_size, 1, fp)
EDIT — MODIFY METADATA
1
Open MP3 file in binary read+write mode
fopen(file, "r+b")
2
Parse frames to find the target frame ID
compare frame_id with "TIT2", "TPE1"...
3
Seek to exact byte offset of frame data
fseek(fp, data_offset, SEEK_SET)
4
Write new value — pad or truncate to fit
fwrite(new_value, size, 1, fp)
5
Close file — audio data untouched
fclose(fp)

What It Does

🎵
ID3v2 Tag Reading
Reads the full ID3v2 header and parses every frame — title, artist, album, year, genre and comment — directly from the binary MP3 file.
✏️
Metadata Editing
Modifies any metadata field by seeking to the exact frame position and writing the new value. No libraries. Pure binary file manipulation in C.
🔒
Audio Data Preserved
All edits happen only inside the ID3 tag block at the start of the file. The actual audio data that follows is never read or modified.
📐
Syncsafe Integer Decoding
ID3v2 stores tag sizes as syncsafe integers — each byte uses only 7 bits. This project correctly decodes these using bitwise shift operations.
🧱
Modular C Architecture
Separate modules for reading, editing, header parsing and frame navigation. Each function does one thing — easy to read, test and extend.
💻
Command-Line Interface
Simple CLI — pass the MP3 filename and the field to edit. No GUI, no dependencies, no installation needed beyond GCC on Linux.

Built With

LANGUAGE
C Programming

FORMAT
ID3v2 Standard
Implements the ID3v2 tag specification — 10-byte header, 4-byte frame IDs, syncsafe integer size encoding and frame data parsing.
I/O
Binary File Handling
fopen, fread, fwrite, fseek and fclose for precise binary file operations. File pointer positioned to exact byte offsets for safe editing.
MEMORY
Pointers & Structs
C structures map to ID3 header and frame layouts. Pointer arithmetic navigates the binary file efficiently without unnecessary buffering.
BITWISE
Bit Manipulation
Syncsafe integers decoded using left shift and OR operations. Frame size correctly reconstructed from four 7-bit encoded bytes.

Compiled with GCC using -Wall -Wextra flags for zero warnings. Developed and tested on Linux WSL environment.
